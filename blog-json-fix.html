You paste a massive API response into your code, hit save, and see this: Uncaught SyntaxError: Unexpected token } in JSON at position 432.

It's the most common error in web development. JSON (JavaScript Object Notation) looks like JavaScript, but it is not JavaScript. It is a language-agnostic data format with zero tolerance for errors.

In this engineering guide, we will move beyond "check your commas" and dive into the specific parsing rules that break your application.

1. Why JSON is So Strict (RFC 8259)

JSON was designed to be parsed easily by any language (Python, Java, C++), not just JavaScript. To make parsing fast and safe, the creators removed all ambiguity.

There are no comments. There are no trailing commas. There are no undefined values. If you deviate by one character, the parser fails to prevent security vulnerabilities.

2. The 3 Most Common Errors

A. The "Trailing Comma" Trap

Modern JavaScript allows trailing commas (commas after the last item). JSON does not. This is the #1 cause of broken config files.

Example: Trailing Comma
// ❌ INVALID JSON
{
  "name": "Udit",
  "role": "Developer", <-- Error Here
}

// ✅ VALID JSON
{
  "name": "Udit",
  "role": "Developer"
}

B. Comments are Forbidden

You cannot use // or /* */ in standard JSON. If you need comments, you must use a superset like JSON5 or YAML, or strip them out before parsing.

3. Single vs. Double Quotes

In JavaScript, 'key' and "key" are identical. In JSON, only double quotes are allowed. This applies to both Keys and Values (unless the value is a number, boolean, or null).

Quote Rules
// ❌ INVALID (Single Quotes)
{
  'name': 'CodeFormats'
}

// ❌ INVALID (No Quotes on Key)
{
  name: "CodeFormats"
}

// ✅ VALID
{
  "name": "CodeFormats"
}

4. The "Hidden" Character Problem

Sometimes your JSON looks perfect, but still fails. This usually happens when copying data from Slack, Jira, or a PDF.

These platforms often insert invisible characters:

  • Smart Quotes: Replacing " with curly quotes “ or ”. JSON parsers do not recognize curly quotes.
  • Zero Width Spaces: Invisible characters (\u200b) attached to the end of strings.
  • Non-Breaking Spaces:   instead of standard spaces.

How to Spot Invisible Characters

In VS Code, enable "Render Control Characters" in settings. Or, paste your JSON into an online validator that highlights illegal characters.

5. Debugging Tools (CLI & VS Code)

Stop scanning lines with your eyes. Use tools.

VS Code Shortcut

Open your file in VS Code. Press Shift + Alt + F (Format Document). If VS Code refuses to format it, look at the bottom right corner for a red error indicator.

The `jq` Command Line Tool

For backend developers, jq is the gold standard. It parses JSON in the terminal and gives meaningful error messages.

Terminal
$ echo '{"foo": "bar",}' | jq .
parse error: Expected another key-value pair at line 1, column 15

Frequently Asked Questions

Can I use functions in JSON? +
No. JSON is a data-interchange format, not a programming language. It supports Strings, Numbers, Booleans, Arrays, Objects, and Null. It does not support functions, undefined, or dates (dates are stored as strings).
Why does JSON.parse() fail on big numbers? +
JavaScript has a maximum safe integer limit (`2^53 - 1`). If your JSON contains a number larger than this (like a database ID), JS will round it, corrupting the data. Solution: Store big numbers as Strings in JSON.

Conclusion

JSON errors are annoying, but usually simple. They almost always boil down to a stray comma, a wrong quote, or a hidden character. Use the tools available to you, and never write JSON manually if you can avoid it—generate it programmatically.
